# Документация меню

Проект содержит структурированную документацию по меню и блюдам. Разбивка и описание хранятся в **Markdown (.md)**; критерии — в `docs/criteria/`, меню и карточки блюд — в `docs/menus/` и `docs/dishes/`.

---

## Назначение папок и файлов

| Путь | Назначение |
|------|------------|
| **docs/criteria/** | Аудитория (семья 5 человек, дети, лактоза), лимиты по времени (завтрак ≤40 мин, ужин до 60/90 мин), бюджет, требования к молочному. Используются при разработке и описании блюд. |
| **docs/menus/** | Разделы меню: завтраки, при необходимости — обеды, ужины и т.д. В каждом файле — список блюд и ссылки на карточки в `dishes/`. |
| **docs/dishes/** | Одна карточка = один .md-файл: название, описание, состав по компонентам, порции, время, аллергены, подача. Связаны с разделами в `menus/`. |

---

## Формат хранения

- Вся новая и разбитая информация хранится в **формате Markdown (.md)**, если не попросят иного.

## Как пользоваться

1. **Критерии и контекст** — смотреть `docs/criteria/audience-and-constraints.md`.
2. **Разделы меню** — `docs/menus/zavtraki.md` (28 завтраков), `docs/menus/obedy.md` (обеденные супы на 3 дня), `docs/menus/poldnik.md` (полдник: ПП-перекусы, часть впрок), `docs/menus/uzhiny.md` (28 ужинов), `docs/menus/nedelya-1.md`–`nedelya-4.md` (меню на 4 недели по дням: завтрак, обед, полдник, ужин).
3. **Закупки на неделю** — `docs/menus/zakupki-na-nedelyu.md` (индекс) и `zakupki-nedelya-1.md` … `zakupki-nedelya-4.md`: продукты с количеством на 5 человек по каждой неделе; из меню недели — ссылка на соответствующий файл закупок.
4. **Конкретное блюдо** — искать карточку в `docs/dishes/` по ссылке из меню или по имени файла.
5. **Разработка и описание меню/блюд** — можно привлекать субагента «Шеф» (`.cursor/agents/shef.md`): он ориентирован на описание меню, блюд и разбивку на компоненты. При генерации меню на неделю Шеф учитывает рейтинги из `docs/data/ratings.json` (см. `docs/data/README.md`).

## Локальный запуск проекта

1. **Требования:** [Node.js](https://nodejs.org/) (например LTS).
2. **Сборка контента** (один раз после клонирования и после правок в `docs/`):
   ```bash
   node scripts/generate-manifest.js
   ```
   Скрипт скопирует меню, блюда и подсказки в `web/content/` и создаст `manifest.json`.
3. **Запуск сервера** из корня проекта:
   ```bash
   node server.js
   ```
   В консоли появится адрес, например: `http://localhost:3000`. Откройте его в браузере.
4. **Альтернатива:** если не используете `server.js`, раздайте папку `web/` любым HTTP-сервером, например:
   ```bash
   npx serve web
   ```
   Затем откройте в браузере указанный адрес (часто `http://localhost:3000` или другой порт).

Сайт по `file://` не работает: контент подгружается через `fetch`, поэтому нужен именно HTTP.

## Веб-приложение и динамическая подгрузка

- **Запуск:** см. раздел [Локальный запуск проекта](#локальный-запуск-проекта) выше.
- **Обновление контента:** после добавления или правки файлов в `docs/` выполните `node scripts/generate-manifest.js`. Скрипт обновит `web/content/` (копия меню, блюд, подсказок и `data/`) и пересоберёт `manifest.json`. Новые и изменённые файлы появятся в приложении без правки кода.
- **Рейтинги блюд:** на карточке блюда можно поставить оценку (1–5 звёзд). Оценки хранятся в localStorage; кнопка «Скачать рейтинги (ratings.json)» выгружает их в файл. Сохраните его в `docs/data/ratings.json` — тогда при следующей генерации меню (Шеф) рейтинги будут учтены (предпочтение блюдам с высоким рейтингом).
- **Мобильные устройства:** вёрстка ориентирована на открытие с телефона: бургер-меню в шапке, крупные области нажатия, горизонтальный скролл таблицы недели, поддержка safe-area (вырезы экрана). Можно добавить на главный экран (PWA-подобное поведение через meta apple-mobile-web-app-capable).
- **Публикация (доступ из любого места):** см. **[docs/publish.md](docs/publish.md)** — пошагово: Netlify, Vercel, Cloudflare Pages, GitHub Pages. В репозитории уже есть `netlify.toml`, `vercel.json` и workflow для GitHub Actions; после подключения к выбранному сервису сайт будет собираться и открываться по постоянному URL с телефона и компьютера.

- Завтраки и ужины из PDF разобраны: 20 + 21 блюдо; добавлены обеды (6 супов на 3 дня), полдники (12 ПП-перекусов: печенье, кисель, смузи, гренки, йогурт, творог, фрукты и т.д.) и новые блюда для 4-й недели. - Обеды: суп варится в воскресенье (на Пн–Ср) и в среду (на Чт–Сб), воскресный обед — по выбору семьи. Окрошка и холодник — на тёплый сезон.
- Полдник: часть перекусов готовится впрок на 2–3 дня (овсяное печенье с финиками/какао, бананово-овсяное печенье, овсяные батончики, кисель); напитки чередуют — какао, травяной чай, кисель. Йогурт БЛ, творог, фрукты — покупное с нарезкой.
- При появлении новых общих правил или аудитории — расширять или добавлять файлы в `docs/criteria/`.

Общая схема документации сохраняется: **исходник → критерии → разделы меню → карточки блюд**.
