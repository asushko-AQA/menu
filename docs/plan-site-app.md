# План сайта / приложения для меню

Документ для субагента «Разработчик» и дальнейшей реализации: как удобно с любого устройства следить за меню, переходя между неделями → день → карточка блюда.

---

## 1. Анализ текущей структуры

### 1.1 Что есть сейчас

| Путь | Назначение | Формат |
|------|------------|--------|
| `docs/menus/nedelya-1.md` … `nedelya-4.md` | Меню на неделю (Пн–Вс) | Таблица: День \| Завтрак \| Обед \| Полдник \| Ужин; ссылки на блюда; блок «Что подготовить по дням» |
| `docs/menus/zavtraki.md` | Список завтраков (28) | Таблица № \| Блюдо \| Карточка \| Время |
| `docs/menus/obedy.md` | Обеды — супы на 3 дня (6 шт.) | Таблица супов + логика Пн–Ср / Чт–Сб, воскресный обед |
| `docs/menus/poldnik.md` | Полдники (12) | Таблица полдников + логика заготовок |
| `docs/menus/uzhiny.md` | Ужины (28) | Таблица ужинов |
| `docs/dishes/*.md` | Карточки блюд (~74 файла) | Заголовок, описание, ингредиенты (таблица), специи, приготовление, состав, порции, аллергены, подача |
| `docs/tips/zagotovki.md` | Заготовки на неделю | Текст + структура по станциям |
| `docs/criteria/`, `docs/source/` | Критерии и исходник | Справочно |

### 1.2 Связи между файлами

- **Неделя → день → блюдо:** в `nedelya-N.md` ячейки таблицы содержат ссылки вида `[Название](../dishes/slug.md)`. Один «день» — это одна строка таблицы (Пн…Вс).
- **Раздел по приёму пищи → блюдо:** в `zavtraki.md`, `obedy.md`, `poldnik.md`, `uzhiny.md` — таблицы со ссылками на `../dishes/имя-файла.md`.
- **Внутри недели:** внизу есть ссылки на следующую/предыдущую неделю и на `obedy.md`, `poldnik.md`, `zagotovki.md`.

### 1.3 Что не хватает для удобной навигации в приложении/сайте

- **Единой точки входа** по «неделям» — сейчас только перекрёстные ссылки в конце каждого `nedelya-N.md`.
- **Явного «сегодня»** — нет привязки дат к неделям (неделя 1 = какие календарные даты).
- **Списка всех блюд по дням в машиночитаемом виде** — только markdown-таблицы, парсинг нужен при сборке или в рантайме.

## 2. Целевая навигация (как должно быть удобно)

Пользователь должен быстро:

1. **Выбрать неделю** (Неделя 1 … Неделя 4) и увидеть сетку Пн–Вс с приёмами пищи.
2. **Открыть конкретный день** (например, среда) и увидеть: завтрак, обед, полдник, ужин — каждое блюдо кликабельно.
3. **Перейти в карточку блюда** — полное описание, ингредиенты, шаги, аллергены, подача.
4. **Вернуться** назад: из карточки — в день или в неделю; из дня — в неделю; общая навигация «Меню» / «Недели» / «Блюда».

Дополнительно полезно:

- **Просмотр по типу приёма:** «Все завтраки», «Все обеды», «Полдники», «Ужины» — списком со ссылками на карточки (как в `zavtraki.md` и др.).
- **Шпаргалка заготовок** — одна страница `zagotovki.md` с возможностью перехода к нужной неделе/дню.
- **Поиск по названию блюда** (опционально, при росте числа карточек).

## 3. Рекомендация: сайт или приложение

- **Предпочтительно: веб-сайт (адаптивный), при необходимости — PWA или оболочка под Android.**

Причины:

- Один код и один источник данных (`docs/`) для десктопа и телефона.
- «С любого места» — достаточно браузера, не нужна установка из магазина.
- Контент уже в markdown: можно генерировать статический сайт (SSG) или подгружать .md и рендерить на клиенте.
- Android-приложение при желании: WebView с тем же URL или PWA «Add to home screen».

Отдельное нативное Android-приложение имеет смысл только если нужны офлайн-режим, уведомления или сложная локальная логика; для «следить за меню и открывать карточки» веба достаточно.

---

## 4. Варианты реализации

### 4.1 Статический сайт (SSG)

- **Идея:** при сборке парсить `docs/menus/*.md` и `docs/dishes/*.md`, генерировать HTML (или JSON + HTML-шаблоны).
- **Плюсы:** быстрая загрузка, можно хостить на GitHub Pages / Netlify / любой хостинг без сервера.
- **Минусы:** при добавлении/правке меню или блюд нужна пересборка (обычно по коммиту или по кнопке).

Инструменты: свой скрипт (Node/Python) + шаблоны, либо 11ty, VitePress, Hugo и т.п. с плагином для markdown.

### 4.2 SPA / статический фронт с загрузкой .md на клиенте

- **Идея:** сайт — одна или несколько HTML-страниц; список недель и меню по дням захардкожен или в одном index.json; текст блюд и разделов подгружается по запросу (например, `docs/dishes/borshch.md`) и рендерится через markdown-парсер (marked, markdown-it и т.д.).
- **Плюсы:** не нужна сборка под контент; можно раздавать `docs/` как статику и грузить .md по путям.
- **Минусы:** нужен либо сервер/хостинг, отдающий файлы из `docs/`, либо предсобранный бандл со всеми .md (или разбивка по чанкам).

### 4.3 Гибрид

- Генерация при сборке: **индекс** (недели, дни, slug блюд) в JSON.
- Страницы недели/дня — статический HTML или один SPA, данные из JSON.
- Карточки блюд: либо сгенерированные HTML, либо загрузка .md по slug и рендер на клиенте.

Рекомендация для старта: **статический сайт с простой сборкой** (скрипт парсит md → JSON + HTML или только HTML), чтобы не зависеть от сервера и дать быструю навигацию «недели → день → карточка».

## 5. Предлагаемая структура URL и экранов

| Эран / URL | Содержание |
|------------|------------|
| `/` или `/menu` | Главная: список недель (Неделя 1 … 4) + ссылки на «Завтраки», «Обеды», «Полдники», «Ужины», «Заготовки». |
| `/menu/nedelya-1` … `/menu/nedelya-4` | Страница недели: таблица 7 дней × приёмы пищи; каждая ячейка — ссылка на блюдо. Блок «Что подготовить по дням» (можно свернуть). |
| `/menu/nedelya-1/sreda` (или `/menu/nedelya-1/3`) | День: один день выбранной недели — завтрак, обед, полдник, ужин; каждая позиция ведёт на карточку. Кнопки «Пред. день» / «След. день». |
| `/dishes/borshch` | Карточка блюда по slug (без .md). Кнопка «Назад» (в день или в неделю). |
| `/zavtraki`, `/obedy`, `/poldnik`, `/uzhiny` | Списки по приёмам пищи (как в текущих md). |
| `/tips/zagotovki` | Шпаргалка заготовок. |

Имена файлов `nedelya-1.md` … `nedelya-4.md` и slug блюд (имя файла без .md) уже задают стабильные идентификаторы — их можно использовать в URL.

## 6. Перегруппировка и дополнение файлов (опционально)

Текущей структуры достаточно, чтобы по путям и ссылкам построить сайт. Ниже — минимальные изменения, которые упростят парсинг и навигацию.

### 6.1 Индекс недель (рекомендуется)

Добавить файл **`docs/menus/README.md`** (или **`docs/menus/index.md`**), который явно перечисляет недели и их назначение:

```markdown
# Меню по неделям

| Неделя | Файл | Описание |
|--------|------|----------|
| 1 | [nedelya-1.md](nedelya-1.md) | Завтрак + обед + полдник + ужин |
| 2 | [nedelya-2.md](nedelya-2.md) | … |
| 3 | [nedelya-3.md](nedelya-3.md) | … |
| 4 | [nedelya-4.md](nedelya-4.md) | Борщ / щи, запечённые и томатные ужины |
| 5 | [nedelya-5.md](nedelya-5.md) | Чечевичный суп, нут, рыбные тефтели, вок, чиа и роллы |
```

При сборке сайта этот файл можно парсить для формирования главной страницы «Выбор недели».

### 6.2 Единый формат ссылок на блюда

Сейчас в меню используются пути вида `../dishes/имя-файла.md`. Для генератора или клиентского роутера удобно:

- Везде использовать **slug** = имя файла без расширения (например `borshch`, `yaichnitsa-glazunya-tosty`).
- В сгенерированном сайте ссылки на карточки — единообразно, например `/dishes/borshch`.

Переименовывать файлы в `docs/dishes/` не обязательно: при сборке достаточно из имени файла отбрасывать `.md` и использовать как slug.

### 6.3 Метаданные в frontmatter (на будущее)

Если появятся генераторы, которые читают YAML frontmatter, можно постепенно добавлять в карточки блюд, например:

```yaml
---
title: Борщ
meal: obedy
time_min: 50
portions: 15
---
```

Это опционально; текущего формата (заголовок H1, таблицы, подзаголовки) достаточно для парсинга.

### 6.4 Что не трогать

- Разделы `docs/menus/zavtraki.md`, `obedy.md`, `poldnik.md`, `uzhiny.md` и структура `docs/dishes/` — оставить как есть.
- Ссылки между документами — относительные, при переносе в другой движок важно сохранять относительные пути или маппинг slug → файл.

---

## 7. Краткий план действий для разработчика

1. **Добавить индекс недель** — `docs/menus/README.md` или `index.md` со ссылками на nedelya-1…4 (см. п. 6.1).
2. **Выбрать способ реализации:** SSG (скрипт или 11ty/VitePress) или SPA с загрузкой .md.
3. **Реализовать маршруты:** главная (недели + разделы) → страница недели → страница дня → карточка блюда; плюс страницы zavtraki/obedy/poldnik/uzhiny и zagotovki.
4. **Парсеры:** 
   - неделя: извлечь таблицу и блок «Что подготовить» из `nedelya-N.md`;
   - день: одна строка таблицы + ссылки на блюда;
   - карточка: заголовок, описание, таблица ингредиентов, секции по подзаголовкам (Приготовление, Аллергены и т.д.).
5. **Вёрстка:** адаптивная, удобные кнопки «Назад», «В день», «В меню недели».
6. **При желании:** PWA (manifest + service worker для офлайна кэширования) или оболочка Android (WebView) с тем же URL.

После этого пользователь сможет с телефона или компьютера открыть сайт, выбрать неделю → день → блюдо и при необходимости просматривать списки по типу приёма пищи и заготовки.
