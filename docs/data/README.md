# Данные для меню

## ratings.json

Рейтинги блюд для учёта при генерации меню.

- **Формат:** `slug` — имя файла карточки без `.md` (например `borshch`, `yaichnitsa-glazunya-tosty`); `rating` — число от 1 до 5; `updatedAt` — дата обновления (ISO).
- **Откуда берётся:** пользователи оценивают блюда в веб-приложении (звёзды на карточке); рейтинги хранятся в localStorage. Кнопка «Скачать рейтинги (ratings.json)» выгружает текущие оценки в файл — его можно сохранить в `docs/data/ratings.json` и закоммитить.
- **Использование при генерации меню:** при составлении недельного меню (субагент «Шеф» или скрипт) читать `docs/data/ratings.json` и:
  - чаще включать блюда с рейтингом 4–5;
  - реже или по запросу включать блюда с рейтингом 1–2;
  - блюда без рейтинга считать нейтральными.

Пример записи в `ratings`:

```json
{ "slug": "borshch", "rating": 5, "updatedAt": "2026-03-15T12:00:00.000Z" }
```
